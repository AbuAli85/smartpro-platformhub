# Protected Handler Operator Visibility

## Purpose
Defines how SmartPRO makes protected-handler exclusions visible to operators and maintainers.

## Current Visibility Mechanisms
- exclusion integrity tests
- review metadata checks
- aging checks
- human-readable exclusion summary

## Why
Operators should be able to answer quickly:
- what exclusions exist?
- who owns them?
- when are they due for review?
- are any near due or overdue?

## Rule
Visibility should remain lightweight, deterministic, and easy to review from code and test outputs.
